# XJOS

XJOS 是一个基于 x86 的 32 位操作系统内核项目，主要用于操作系统原理学习、内核实现实践和底层调试研究。

这个项目并不是完全从零闭门造车完成的，而是在学习和参考开源操作系统项目实现思路的基础上，结合自己的理解持续补全、重构和完善内核功能。
> 本项目参考了 [onix](https://github.com/StevenBaby/onix) 开源项目, 感谢作者提供的教学思路，使我能够实现从零到一开发操作系统内核

## 项目目标

- 学习并实践操作系统启动、内存管理、进程调度、文件系统和设备驱动等核心机制
- 打通从 Bootloader 到用户态程序运行的完整链路
- 在参考开源项目的基础上，逐步演进出更完整、更可维护的内核实现
- 为后续网络子系统和协议栈实现打基础

## 当前已实现功能

### 1. 启动与体系结构基础

- Bootloader 与 Loader，支持从引导阶段加载内核
- 进入保护模式并完成基础运行环境初始化
- 构建 GDT、TSS 和中断描述符表
- 建立分页机制与内核页表映射
- 支持内核态与用户态切换

### 2. 中断、时钟与定时器

- 中断子系统初始化与异常处理框架
- 8259A/PIT 时钟初始化
- 内核时钟中断与 jiffies 驱动的时间管理
- 软件定时器与任务休眠/唤醒机制
- RTC 时间相关支持

### 3. 进程、调度与同步

- 任务控制块、内核线程与用户进程管理
- 基于红黑树的 CFS 完全公平调度器
- 支持 `fork`、`execve`、`exit`、`waitpid` 等进程生命周期管理
- 支持进程组、会话、前台进程组等基础终端作业控制能力
- 支持信号、`alarm` 等机制
- 实现互斥锁、自旋锁、信号量等同步原语

### 4. 内存管理

- 物理内存探测与页级内存分配
- 页目录、页表管理与虚拟地址映射
- 内核堆分配器（Arena）
- 用户进程地址空间管理
- `fork` 场景下的写时复制（COW）
- `brk`、`mmap`、`munmap` 等基础内存接口

### 5. 文件系统与系统调用

- 系统调用分发框架
- Minix 文件系统基础支持
- Super Block、inode、buffer cache、目录项缓存等核心结构
- 支持挂载、卸载、格式化等基础文件系统操作
- 支持 `open`、`close`、`read`、`write`、`lseek`、`stat`、`mkdir`、`link`、`unlink` 等常用文件接口
- 支持管道、dup/dup2 等基础 Unix 风格文件描述符机制

### 6. 用户态支持

- 支持 ELF 可执行文件装载
- 支持 `/bin/init` 作为用户态初始进程启动
- 支持 BusyBox 风格命令组织方式
- 已具备基础 Shell 运行能力
- 支持用户程序通过系统调用访问内核服务

### 7. 设备驱动与外设

- 控制台与 TTY 子系统
- 键盘输入处理
- 串口调试输出
- IDE 磁盘驱动与分区识别
- RAM Disk 支持
- Floppy 驱动
- PCI 总线初始化
- FPU 状态管理
- SB16 声卡驱动的基础支持

## 当前项目状态

目前 XJOS 已经具备一个教学型操作系统内核的核心骨架，打通了以下完整链路：

`Bootloader -> 内核初始化 -> 中断/时钟 -> 内存管理 -> 调度 -> 文件系统 -> 系统调用 -> 用户态 init -> Shell`

```bash
make qemu
```

## 说明

XJOS 是一个持续演进中的学习型内核项目。项目中的部分设计思路来自对已有开源操作系统项目的学习与吸收，但当前实现并非简单复刻，而是在原有基础上针对进程管理、调度、内存、文件系统和用户态支持做了进一步扩展、修正和完善。
